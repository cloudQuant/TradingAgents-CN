✅ bond_info_cm 字段显示顺序已优化

问题：code, endpoint, source 这三个字段显示在最前面
原因：这些是元数据字段，值都相同，影响业务数据查看

解决：将这三个字段移到最后

优化前：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. code            ← 元数据
2. endpoint        ← 元数据
3. source          ← 元数据
4. 债券简称
5. 债券代码
6. 发行人/受托机构
...

优化后：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 债券简称         ⭐ 业务数据前置
2. 债券代码         ⭐
3. 发行人/受托机构   ⭐
4. 债券类型         ⭐
5. 发行日期         ⭐
6. 最新债项评级      ⭐
...
N-2. code          ← 移到最后
N-1. endpoint      ← 移到最后
N.   source        ← 移到最后

修改文件：
  app/routers/bonds.py (+9行)
  
实现逻辑：
  在返回字段信息前，对 bond_info_cm 集合
  重新排序字段列表，将元数据字段移到最后

影响：
  ✓ 只影响字段显示顺序
  ✓ 不改变数据存储
  ✓ 不影响查询和过滤
  ✓ 向后兼容

### 获取数据的API接口、字段等

#### 债券基础信息-中国外汇交易中心

**接口**: `bond_info_detail_cm`

**目标地址**: https://www.chinamoney.com.cn/chinese/zqjc/?bondDefinedCode=egfjh08154

**描述**: 中国外汇交易中心暨全国银行间同业拆借中心-数据-债券信息-信息查询-债券详情

**限量**: 单次返回指定债券的详细信息

**输入参数**

| 名称     | 类型  | 描述                                                 |
|--------|-----|----------------------------------------------------|
| symbol | str | symbol="19万林投资CP001"; 通过 ak.bond_info_cm() 查询 债券简称 |

**输出参数**

| 名称    | 类型     | 描述  |
|-------|--------|-----|
| name  | object | 字段名 |
| value | object | 字段值 |

**接口示例**

```python
import akshare as ak

bond_info_detail_cm_df = ak.bond_info_detail_cm(symbol="19万林投资CP001")
print(bond_info_detail_cm_df)
```

**典型字段说明**

主要包含以下类型的字段：
- **基本信息**：债券全称(bondFullName)、债券简称(bondName)、债券代码(bondCode)、ISIN代码(isinCode)
- **发行信息**：发行人、发行规模、发行价格、发行日期
- **期限信息**：起息日、到期日、债券期限
- **利率信息**：票面利率、付息频率、付息方式、基准利率
- **评级信息**：债项评级、主体评级、评级机构
- **条款信息**：含权条款、回售条款、赎回条款
- **其他信息**：承销商、担保情况、用途说明

### 实现要点

1. **后端实现**：
   - 扩展 `BondDataService` 服务类
   - 实现详细信息获取、存储、更新逻辑
   - 实现批量获取（从债券查询集合获取债券简称列表）
   - 实现增量更新（识别缺失详情的债券）
   - 实现并发控制（避免API限流）
   - 实现数据解析和格式化（name-value结构转换为字典）
   - 实现文件导入和远程同步功能

2. **前端实现**：
   - 参考债券查询页面的布局
   - 实现数据概览组件：
     - 总记录数
     - 覆盖债券数
     - 最近更新时间
     - 数据完整度统计
   - 实现数据列表组件：
     - 支持分页（默认每页50条）
     - 支持排序（按更新时间、发行日期等）
     - 支持筛选（按债券类型、评级等）
     - 支持字段展示/隐藏
   - 实现更新数据组件：
     - 单个更新：输入债券简称
     - 批量更新配置：
       - 数据源选择（从债券查询集合获取或手动输入）
       - 批量大小、并发数、延迟时间
     - 增量更新：自动识别缺失数据
   - 实现详情查看：展示所有字段的详细信息
   - 增加数据可视化：
     - 数据完整度统计图
     - 评级分布图
     - 期限分布图

3. **数据模型**：
   ```python
   {
       "债券代码": str,  # 唯一标识
       "债券简称": str,
       "债券全称": str,
       "查询代码": str,
       "发行人": str,
       "发行规模": float,
       "发行价格": float,
       "发行日期": str,
       "起息日": str,
       "到期日": str,
       "债券期限": str,
       "票面利率": float,
       "付息频率": str,
       "付息方式": str,
       "债项评级": str,
       "主体评级": str,
       "评级机构": str,
       "原始数据": dict,  # 保存所有name-value对
       "更新时间": datetime,
       "数据来源": "chinamoney"
   }
   ```

4. **注意事项**：
   - API调用频率控制：建议每次请求间隔0.5-1秒
   - 批量更新时需要显示详细进度（当前债券、进度百分比、预计剩余时间）
   - 错误处理：网络异常、API限流、债券不存在等
   - 数据验证：检查必要字段是否存在
   - name-value结构需要转换为易于查询的字典结构
   - 建议同时保存原始数据和解析后的数据

5. **批量更新配置**：
   - 批量大小：建议10-50只债券为一批
   - 并发数：建议1-2（避免被限流）
   - 延迟时间：建议0.5-1秒/次
   - 数据源：
     - 从债券查询集合获取：可以获取最新的债券列表
     - 手动输入：适合更新特定债券
     - 增量更新：只更新缺失详情的债券

6. **性能优化**：
   - 后端：使用bulk_write进行批量upsert
   - 前端：使用虚拟列表或分页，避免一次渲染太多数据
   - 缓存：可以考虑缓存常用债券的详细信息
   - 索引：在债券代码、债券简称、更新时间字段上建立索引

### 获取数据的API接口、字段等

#### 可转债强赎-集思录

**接口**: `bond_cb_redeem_jsl`

**目标地址**: https://www.jisilu.cn/data/cbnew/#redeem

**描述**: 集思录可转债-强赎

**限量**: 单次返回所有数据

**输入参数**

| 名称  | 类型  | 描述  |
|-----|-----|-----|
| -   | -   | -   |

**输出参数**

| 名称    | 类型      | 描述      |
|-------|---------|---------|
| 代码    | object  | -       |
| 名称    | object  | -       |
| 现价    | float64 | -       |
| 正股代码  | object  | -       |
| 正股名称  | object  | -       |
| 规模    | float64 | 注意单位: 亿 |
| 剩余规模  | float64 | -       |
| 转股起始日 | object  | -       |
| 最后交易日 | object  | -       |
| 到期日   | object  | -       |
| 转股价   | float64 | -       |
| 强赎触发比 | int64   | 注意单位: % |
| 强赎触发价 | float64 | -       |
| 正股价   | float64 | -       |
| 强赎价   | float64 | -       |
| 强赎天计数 | object  | -       |
| 强赎条款  | object  | -       |
| 强赎状态  | object  | -       |

### 实现要点

1. **后端实现**：
   - 实现强赎数据获取
   - 实现强赎状态解析
   - 实现强赎预警计算

2. **前端实现**：
   - 强赎状态分类显示
   - 强赎倒计时提醒
   - 强赎风险评估
   - 数据可视化：
     - 强赎状态分布饼图
     - 强赎天计数进度条
     - 强赎触发价与正股价对比

3. **数据模型**：
   ```python
   {
       "代码": str,
       "名称": str,
       "现价": float,
       "正股代码": str,
       "正股名称": str,
       "规模": float,  # 亿
       "剩余规模": float,
       "转股起始日": str,
       "最后交易日": str,
       "到期日": str,
       "转股价": float,
       "强赎触发比": int,  # %
       "强赎触发价": float,
       "正股价": float,
       "强赎价": float,
       "强赎天计数": str,
       "强赎条款": str,
       "强赎状态": str,
       "强赎风险等级": str,  # 计算得出：高/中/低
       "更新时间": datetime
   }
   ```

4. **注意事项**：
   - 强赎天计数格式：如"25/15 | 30"表示已满足25天/需要15天，总计30天
   - 强赎状态包括：已公告强赎、接近强赎、无强赎风险等
   - 需要计算强赎风险等级
